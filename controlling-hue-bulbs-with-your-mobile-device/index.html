Controlling Hue Bulbs With Your Mobile Device

21 March 2015

In the past few years, I have become fascinated with the Internet of Things. The idea that devices can be connected and interact with one another across a wireless network. According to Wikipedia, the internet of things (IoT) is: " the network of physical objects or "things" embedded with electronics, software, sensors and connectivity to enable it to achieve greater value and service by exchanging data with the manufacturer, operator and/or other connected devices."

With this capability, the possibilites are endless! I'll admit, while this is incredibly awesome, when it comes to thinking about the security and personal privacy challenges this creates, this is a little scary!

Nevertheless, During my time at the Turing School of Software and Design, I had the opportunitiy to take an elective about Hue Bulbs. Hue Bulbs are wireless led light bulbs manufactured by Phillips. After spending a few weeks learning about how these work. I decided to tap into the IoT and put my hacking, api, and rails skills to the test. The end goal was a simple app that will allow you to control the Hue Bulbs with your mobile device.

For the App I used Twilio, Ruby on Rails, the 'Hue' gem and ngrok. The overall setup time for the app was about 2 hours. Below is an outline of the steps I took to create the app.

  • I started by generating a new rails application which I named HueBulbApp.

  • Next I added the Hue gem in the gemfile

  • I then generated a controller which I called 'Twilio' and set up a route that will allow Twilio to post information to the create action on this Controller.

  • I set up a twilio account and generated a phone number that would send an HTTP Post request via sms to my app route.

  • The next step was to set up a ruby class that contains the code for connecting with the Hue Bulbs. I named this 'HueHandler' as its job was to handle all of the communication with the hue bulbs. I set up a method called 'response' that accepted the information from the sms as a parameter. Based on that information, the response method would then give a command to the hue bulbs.

  • Its important to note that because of the sometimes intermittent wireless connection, it was important to rescue any connection errors when trying to connect with the hue bulbs. Without doing this, the app would break immediately if it was unable to connect to the bulb router.

  • I then setup the create action of the twilio controller to call the HueHandler Response method passing in the params which is the infromation coming in from the sms message via Twilio.

  • Lastly, because there is no user interface on the application side, I used Ngrok to expose my app to the web in order recieve the information coming in from Twilio as oppsed to using a cloud application platform such as Heroku.

  • In the end, the HueBulbApp was capable of communicating with Hue bulbs in 2 ways:

    1. Turning the bulbs on and off by texting "on" or "off" text to the twilio number
    2. Changing the bulb hue, i.e. turing the color to green, pink or blue by texting "Pink", "Blue", or "Green".

  • All of the code can be found at: https://github.com/scottalexandra/HueBulpApp
